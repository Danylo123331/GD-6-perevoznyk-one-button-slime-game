# Лабораторна робота №6 — Створення «однокнопкової гри»

---

## "Hurry to change your shape!"
**"Hurry to change your shape!"** - це динамічна однокнопкова 2D аркада на точність.

### Опис механіки в одному реченні:
> Гравець керує геометричною фігурою, яка сама рухається вперед, і за допомогою всього одного кліку мишки трансформує свою форму, щоб проскочити крізь перешкоди, а на кнопку **Tab** можна змінити сам режим управліня.

---

## Реалізований функціонал & Core Loop

1. **Ігровий цикл (Core Loop):** Старт із головного меню - автоматичний рух із набором очок за кожну пройдену стіну - проїграш при врізанні - виведення рекорду і кнопка рестарту для нового кола.
2. **Керування однією кнопкою:** Сама гра повністю ведеться через клік мишки (одна дія). Кнопка `Tab` винесена окремо суто для зміни режиму управліня.
3. **Елементи інтерфейсу (UI) та логіка сцени:**
   * **Головне меню:** Сцена, з якої все запускається, є вкладки для чеку статистики та налаштувань.
   * **Меню Паузи (Pause Menu):** Повністю стопає ігровий процес через `Time.timeScale = 0`, працюють кнопки *Resume*, *Restart* та *Exit*.
   * **Екран Програшу (Game Over):** Вилітає при колізії, показує поточний рахунок і найкращий результат (`Best Score`), який зберігається через `PlayerPrefs`.
4. **Кастомізація дизайну:**
   * Повністю переробив дефолтний візуальний стиль шаблону. Зробив кастомний мінімалістичний дизайн: контрастна фіолетова матова дорога, чорний фон і яскраві червоні елементи гравця та кнопок інтерфейсу.
5. **Оптимізація коду:**
   * Переписав логіку в скрипті `MenuSelect.cs`, додав перевірки на `null` для всіх об'єктів. Тепер у консолі нуль помилок типу `UnassignedReferenceException`, навіть якщо якісь елементи інтерфейсу видалені або вимкнені.

## Інструкція із запуску
1. Завантажити репозиторій.
2. Відкрити папку `slime-game` через Unity Hub.
3. Запустити стартову scenic-сцену з папки `Assets/Scenes/`.
